Create and train workers by simply posting text instructions.

Create tools by simply writing shell scripts.

Or set your LLM agent loose on your org's documentation to instruct the workers and create the tools.

The python code (on the order of 200 lines) can be copied to a company's computer, or a person's Raspberry Pi, with slight modifications and run.

A minimal html chat client is shown. Something like it could be incorporated into a company's website.

The workers are token-wise inexpensive. Their specialization makes for small context windows and they work nicely with cheap LLMs.

Each worker stores all messages it sends or receives in per-user-account folders.
So it has a complete chronological transcript, a dossier as it were, for each user.

The possibilities for such a paper trail are considerable.
